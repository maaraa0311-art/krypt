# KRYPT — Веб хөгжүүлэлтийн студи

Жижиг бизнес, стартап, хувийн брэндэд зориулсан веб хөгжүүлэлтийн студийн сайт + **хэрэглэгчийн портал** (нэвтрэх, захиалга өгөх, явц хянах).

## Файлууд

| Файл | Зориулалт |
|------|-----------|
| `index.html` | Үндсэн танилцуулга сайт (landing) |
| `login.html` | Нэвтрэх / бүртгүүлэх хуудас |
| `dashboard.html` | Хэрэглэгчийн портал + админ удирдлага |
| `portal.css` | login/dashboard-ийн хуваалцсан загвар |
| `backend.js` | Backend абстракц (API / Supabase / demo автоматаар сонгоно) |
| `supabase-config.js` | **Backend тохиргоо** (API хаяг эсвэл Supabase түлхүүр) |
| `server/` | **Node.js + PostgreSQL backend сервер** |
| `schema.sql` | Supabase ашиглах тохиолдлын бүтэц (заавал биш) |
| `krypt-logo.png` | Брэндийн лого |

## 🔧 Backend сонголт

`supabase-config.js` доторх утгаас хамаарч **3 горим** автоматаар сонгогдоно:

1. **Өөрийн PostgreSQL сервер** — `API_BASE_URL` тохируулсан бол (← **одоогийн сонголт**). `server/README.md`-г үзнэ үү.
2. **Supabase** — Supabase түлхүүр тохируулсан, API_BASE_URL хоосон бол.
3. **Demo (localStorage)** — аль нь ч тохируулаагүй бол. Тохиргоо хэрэггүй, гэхдээ өгөгдөл зөвхөн тухайн хөтөч дээр.

### → PostgreSQL сервер тохируулах
Бүрэн заавар: **[`server/README.md`](server/README.md)**. Товчхондоо:
```bash
cd server
npm install
cp .env.example .env     # DATABASE_URL, JWT_SECRET тааруул
npm run init-db          # хүснэгт үүсгэх
npm start                # → http://localhost:4000/api
```
Дараа нь сайтаар бүртгүүл (анхны хэрэглэгч = админ).

## ⚙️ (Нөөц хувилбар) Supabase тохируулах (~5 минут)

> Зөвхөн PostgreSQL серверийн оронд Supabase ашиглах бол. `API_BASE_URL`-г хоосон болгоно.

### 1. Project үүсгэх
1. [supabase.com](https://supabase.com) → бүртгүүлэх (үнэгүй).
2. **New project** → нэр, нууц үг өгч үүсгэнэ.

### 2. Өгөгдлийн сан бэлдэх
1. Зүүн талын цэснээс **SQL Editor** → **New query**.
2. `schema.sql` файлын бүх агуулгыг хуулж буулгаад **Run** дарна.

### 3. Түлхүүрээ хуулах
1. **Project Settings** (доод зүүн) → **Data API**.
2. Дараах 2 утгыг хуулна:
   - **Project URL**
   - **API Keys → `anon` `public`**
3. `supabase-config.js` файлыг нээгээд эхний 2 утгыг солино:
   ```js
   const SUPABASE_URL = 'https://xxxx.supabase.co';
   const SUPABASE_ANON_KEY = 'eyJhbGci...';
   ```

### 4. (Зөвлөмж) Имэйл баталгаажуулалт унтраах
Туршихад хялбар болгохын тулд:
**Authentication → Sign In / Providers → Email** → **Confirm email**-ийг унтраана.
(Бодит ашиглалтад асаалттай үлдээвэл аюулгүй.)

### 5. Өөрийгөө админ болгох
1. Сайт дээрээсээ (`login.html`) энгийн хэрэглэгчээр **бүртгүүлнэ**.
2. Supabase → **SQL Editor** дээр дараахыг (имэйлээ тааруулж) Run хийнэ:
   ```sql
   update public.profiles set role = 'admin'
   where id = (select id from auth.users where email = 'maaraa0311@gmail.com');
   ```
3. Дахин нэвтрэхэд **админ** эрхээр бүх захиалгыг харж, статус шинэчилнэ.

## Ажиллах урсгал

- **Хэрэглэгч**: бүртгүүлэх → нэвтрэх → захиалга өгөх → статусаа real-time хянах.
- **Админ (та)**: бүх захиалгыг харах → статус сонгох (Шинэ → Хянаж байна → Хийгдэж байна → Хүлээлгэн өгсөн / Цуцлагдсан) → хэрэглэгчийн дэлгэц шууд шинэчлэгдэнэ.

Статусын нэрийг `supabase-config.js` доторх `ORDER_STATUS`-аас өөрчилж болно.

## Ажиллуулах

Статик сайт тул шууд хостинг дээр тавьж болно (GitHub Pages, Vercel, Netlify). Локалд:

```bash
npx serve .
```

> Тэмдэглэл: нэвтрэлт зөв ажиллахын тулд `index.html`-г файлаас шууд нээхийн оронд **локал сервер** эсвэл хостингоор дамжуулж нээхийг зөвлөнө.

## Аюулгүй байдал

- `anon` key нийтэд харагдахад **аюулгүй** — өгөгдлийг **Row Level Security (RLS)** хамгаална (`schema.sql`-д тохируулсан).
- Хэрэглэгч зөвхөн өөрийн захиалгыг харна; статусыг зөвхөн админ өөрчилнө.
